### **Updated Summary of Features (E-Commerce Project) in Hinglish**

#### **1. Teen Roles ke Users**
- **Explanation**: Project mein teen tarah ke users honge—**Customer**, **Retailer**, aur **Admin**. Har ek ka alag role hoga aur unke hisaab se features ka access milega.
  - **Customer**: Ye normal buyers hain jo products khareedenge, cart aur wishlist use karenge, aur reviews denge. Inka kaam main domain `example.com` pe hoga.
  - **Retailer**: Ye bulk buyers hain jo minimum quantity ke saath products khareedenge, bulk discounts payenge, aur apna alag cart aur wishlist use karenge. Inka interface subdomain `retailer.example.com` pe hoga.
  - **Admin**: Ye site ka owner hai jo sab kuch manage karega—products, categories, coupons, orders, reviews, aur stock alerts. Admin ka dashboard subdomain `admin.example.com` pe hoga.

#### **2. Authentication (Login/Signup)**
- **Explanation**: Har user (Customer, Retailer, Admin) ko email aur password se login karna hoga.
  - **Register**: Customer aur Retailer apne respective domains (`example.com` aur `retailer.example.com`) se register karenge. Admin ko manually database mein add karna padega.
  - **Login**: User email aur password se login karega, backend role fetch karega, aur role ke basis pe frontend user ko sahi subdomain pe redirect karega (e.g., admin → `admin.example.com`).
  - **Forgot Password**: Sab users ke liye available hai—email pe reset link milega.
  - **Profile Section**: Users apna name, email, address, mobile update kar sakte hain.

#### **3. Products aur Categories**
- **Explanation**: Admin products aur categories add karega, aur ye customer aur retailer ke liye alag-alag honge.
  - **Customer Products**: Normal products jo customer khareed sakte hain (e.g., "Charger - ₹500").
  - **Retailer Products**: Bulk products jo sirf retailer khareed sakte hain (e.g., "Charger - ₹450, min 20 units").
  - **Features**: Har product ke paas name, price, stock, description, image, aur stock status (in_stock/out_of_stock) hoga.
  - **Filters**: Category aur price range ke basis pe products filter karne ka option hoga.

#### **4. Discounts aur Coupons**
- **Explanation**: Discounts aur coupons se price kam hoga.
  - **Normal Discount**: Customer aur retailer dono ke liye (e.g., "Buy 10, get 10% off").
  - **Bulk Discount**: Sirf retailer ke liye (e.g., "50 units pe 15% extra off").
  - **Coupons**: Admin coupons generate karega jo customer, retailer, ya dono ke liye ho sakte hain. Specific users ke liye bhi coupons banaye ja sakte hain (e.g., sirf kuch retailers ke liye).

#### **5. Cart aur Wishlist**
- **Explanation**: Shopping ke liye cart aur wishlist dono honge.
  - **Customer**: Apne cart mein products add karega, wishlist mein save karega.
  - **Retailer**: Apna alag cart aur wishlist use karega, lekin minimum quantity check hoga cart mein.
  - **Login Zaroori**: Cart aur wishlist use karne ke liye login karna padega.

#### **6. Orders**
- **Explanation**: Customer aur Retailer orders place karenge, cash on delivery.
  - **Customer Order**: Kitni bhi quantity khareed sakta hai, address aur mobile number daalna hoga.
  - **Retailer Order**: Minimum quantity ke saath bulk order, bulk discount apply hoga.
  - **Order History**: User apne orders dekh sakta hai (status: pending, shipped, delivered).
  - **Order Cancellation**: User cancel request daal sakta hai, admin approve karega.

#### **7. Reviews**
- **Explanation**: Sirf customer products pe feedback de sakta hai.
  - Rating (1-5 stars) aur comment daalega.
  - Admin saare reviews dekhega, edit ya delete kar sakta hai.

#### **8. Stock Management**
- **Explanation**: Stock ka control admin ke paas hoga.
  - Jab stock kam ho jayega (stock_alert_threshold se neeche), to admin ko email alert milega.
  - Stock_Alerts table mein record rakha jayega aur admin dashboard pe dikhega.

#### **9. Admin Dashboard**
- **Explanation**: Admin ke liye full control panel hoga.
  - **Products aur Categories**: Add, edit, delete kar sakta hai (customer ya retailer ke liye).
  - **Coupons**: Generate aur manage kar sakta hai (target_role aur specific users ke liye).
  - **Orders**: Dekhega, status update karega (e.g., "shipped").
  - **Reviews**: Manage karega (edit/delete).
  - **Stock Alerts**: Low stock products ki list dekhega aur manually email bhej sakta hai.

#### **10. Email Notifications**
- **Explanation**: Email se updates milega.
  - **Order Place**: Jab order place hota hai, to admin ko email (customer ya retailer ka order).
  - **Stock Low**: Jab stock kam hota hai, to admin ko alert.
  - **Forgot Password**: Reset link email pe.

#### **11. Filters aur Search**
- **Explanation**: Products ko easily dhoondhne ke liye.
  - **Filters**: Category aur price range se filter kar sakte hain.
  - **Search**: Product name ya description se search kar sakte hain.

### **Step-by-Step Features with Explanation**
Ab har feature ko step-by-step explain karte hain taaki koi bhi naya developer easily samajh sake:

1. **User Registration (Signup)**
   - **Customer**: Main domain `example.com` pe register karega.
   - **Retailer**: Subdomain `retailer.example.com` pe register karega.
   - **Admin**: Manually database mein add kiya jayega.
   - **Process**:
     1. User name, email, phone number, aur password bhejega.
     2. Backend email aur phone number ki uniqueness check karega.
     3. Role assign hoga (customer/retailer).
     4. Password hash karke database mein save hoga.

2. **User Login**
   - **Process**:
     1. User email aur password bhejega.
     2. Backend credentials verify karega.
     3. Role fetch karega (customer/retailer/admin).
     4. JWT token generate hoga aur frontend ko milega.
     5. Role ke basis pe redirect hoga (e.g., admin → `admin.example.com`).

3. **Profile Management**
   - **Process**:
     1. User apna name, email, address, mobile update kar sakta hai.
     2. Backend token se user verify karega aur update save karega.

4. **Product Browsing**
   - **Customer**: Bina login ke products dekh sakta hai (`GET /api/products/customer`).
   - **Retailer**: Login ke baad apne products dekh sakta hai (`GET /api/products/retailer`).
   - **Filters**: Category aur price range se filter kar sakte hain.
   - **Search**: Product name ya description se search kar sakte hain.

5. **Discounts**
   - **Normal Discount**: Customer aur retailer ke liye (e.g., 10 units pe 10% off).
   - **Bulk Discount**: Retailer ke liye (e.g., 50 units pe 15% off).
   - **Coupons**: Checkout pe coupon code apply kar sakte hain (agar eligible hai to).

6. **Cart**
   - **Customer**: Normal cart, kitni bhi quantity add kar sakta hai.
   - **Retailer**: Cart mein minimum quantity check hoga.
   - **Process**:
     1. User product add karega.
     2. Backend role check karega aur retailer ke liye rules apply karega.

7. **Wishlist**
   - **Process**:
     1. User product ko wishlist mein save karega.
     2. Login zaroori hoga.

8. **Order Placement**
   - **Customer**: Checkout pe address aur mobile daalega.
   - **Retailer**: Bulk order, minimum quantity aur bulk discount ke saath.
   - **Process**:
     1. User cart se checkout karega.
     2. Backend discount apply karega.
     3. Order save hoga aur stock update hoga.
     4. Admin ko email notification jayega.

9. **Order Tracking aur Cancellation**
   - **Order History**: User apne orders dekh sakta hai (status ke saath).
   - **Cancellation**: User cancel request bhejega, admin approve karega.
   - **Process**:
     1. User cancel request bhejega.
     2. Admin approve karega, stock wapas add hoga.

10. **Reviews**
    - **Customer**: Product pe rating aur comment daalega.
    - **Admin**: Reviews ko edit ya delete kar sakta hai.
    - **Process**:
      1. Customer review submit karega.
      2. Admin dashboard se manage karega.

11. **Stock Management**
    - **Process**:
      1. Order place hone pe stock update hoga.
      2. Stock `stock_alert_threshold` se kam hone pe admin ko email jayega.
      3. Admin dashboard pe low stock list dikhegi.

12. **Admin Dashboard**
    - **Features**:
      - Products aur categories manage karega.
      - Coupons generate karega (target_role ya specific users ke liye).
      - Orders ka status update karega.
      - Reviews manage karega.
      - Stock alerts dekhega.

13. **Email Notifications**
    - **Types**:
      - Order place hone pe admin ko.
      - Stock low hone pe admin ko.
      - Forgot password ke liye reset link.
