### Purra Electronics E-Commerce - Project Summary (Hinglish)

Ye summary **Purra Electronics E-Commerce** platform ke liye hai, jo 26 Sequelize models ke saath aligned hai. Isme core features, flow, aur tables ka data examples ke saath diya gaya hai taaki new user ko samajh aaye ki platform kaise kaam karta hai aur features ka flow kya hai. Hinglish mein likha hai, concise aur clear.

---

### Core Features & Flow

#### 1. User Roles
**Models**: Users, Owners, Stores  
**What**: Alag-alag roles ke users ke liye customized experience.  
**Flow**:  
- **Customers**: Retail price pe products khareedte hain, wishlist banate hain, reviews dete hain.  
  - **Example**: User "Rahul" iPhone 15 ko ₹79,999 pe khareedta hai, wishlist mein Samsung TV add karta hai (Users, Wishlists, WishListItems).  
- **Retailers**: Bulk orders ke liye special discounts (e.g., 50 units pe 10% off).  
  - **Example**: Retailer "Amit" 100 iPhone 15 order karta hai, 10% discount milta hai (Users, Orders, DiscountRules).  
- **Admins**: System control karte hain—users ban/unban, products/categories add/edit, stock alerts manage.  
  - **Example**: Admin "Priya" low stock alert set karti hai jab iPhone 15 ka stock 5 bachta hai (Users, StockAlerts).  
- **Owners**: Multi-vendor stores aur products manage karte hain.  
  - **Example**: Owner "Vikram" apna store "TechHub" banata hai aur iPhone 15 add karta hai (Owners, Stores, Products).  
**Validation**: JWT middleware se role-based access control hota hai. Retailers ke orders MOQ (minimum order quantity) ke saath validate hote hain.

#### 2. Authentication System
**Models**: Users  
**What**: Secure login aur user management.  
**Flow**:  
- **Login**: Email/password se JWT token milta hai, jo protected APIs ke liye use hota hai.  
  - **Example**: Rahul email "rahul@example.com" aur password se login karta hai, token milta hai (Users).  
- **Registration**: Single domain pe role select karke register (customer/retailer). Retailers ke liye optional business verification.  
  - **Example**: Amit retailer role select karta hai, business details daalta hai (Users).  
- **Password Reset**: Email pe reset link bheja jata hai.  
  - **Example**: Rahul "forgot password" click karta hai, link email pe aata hai (Users).  
- **Profile Management**: Name, email, phone, photo update karo.  
  - **Example**: Rahul apna phone number update karta hai to +91-9876543210 (Users).  
**Validation**: JWT token har API call pe check hota hai, email format validated hota hai.

#### 3. Product Catalog
**Models**: Categories, Brands, Products, ProductVariants, ProductMedia, ProductMediaURL, Attributes, AttributeValues, VariantAttributeValues, StockAlerts  
**What**: Products aur unke details ka management.  
**Flow**:  
- **Categories**: Multi-level categories (e.g., Electronics → Mobiles).  
  - **Example**: Category "Mobiles" ke under sub-category "Smartphones" (Categories).  
- **Brands**: Admins brands add/edit karte hain.  
  - **Example**: Admin brand "Apple" add karta hai (Brands).  
- **Products & Variants**: Products ke variants alag price/stock ke saath.  
  - **Example**: Product "iPhone 15" ke variants—128GB Black (₹79,999, 50 stock), 256GB Blue (₹89,999, 30 stock) (Products, ProductVariants).  
- **Attributes**: Product properties define karo (e.g., Color: Black, Storage: 128GB).  
  - **Example**: iPhone 15 ka variant 128GB Black ka attribute "Color: Black" set hota hai (Attributes, AttributeValues, VariantAttributeValues).  
- **Media**: Images/videos S3 pe store hote hain.  
  - **Example**: iPhone 15 ke liye 3 images upload hote hain (ProductMedia, ProductMediaURL).  
- **Stock Alerts**: Low stock pe automatic alerts.  
  - **Example**: iPhone 15 128GB ka stock 5 bachta hai, alert trigger hota hai (StockAlerts).  
**Validation**: Variants attributes se linked hote hain, stock real-time update hota hai, media URLs valid hote hain.

#### 4. Shopping Features
**Models**: Carts, CartItems, Wishlists, WishListItems, Products, ProductVariants  
**What**: Shopping ka seamless experience.  
**Flow**:  
- **Cart**: Products/variants add karke quantity adjust karo.  
  - **Example**: Rahul iPhone 15 128GB Black × 2 add karta hai, total ₹1,59,998 (Carts, CartItems).  
- **Wishlist**: Products save karo for later.  
  - **Example**: Rahul Samsung TV wishlist mein add karta hai, baad mein cart mein daalta hai (Wishlists, WishListItems).  
- **Bulk Pricing**: Retailers ke liye tiered discounts.  
  - **Example**: Amit 100 iPhone 15 order karta hai, 10% discount apply hota hai (CartItems, DiscountRules).  
- **Search & Filters**: Name, category, brand, price ke basis pe search.  
  - **Example**: Rahul "iPhone" search karta hai, filters lagata hai (Products, Categories, Brands).  
**Validation**: Cart items MOQ ke saath validate hote hain, wishlist se cart transfer smooth hota hai.

#### 5. Order Processing
**Models**: Orders, OrderItems, Payments, Addresses, ProductVariants  
**What**: Orders place aur track karo.  
**Flow**:  
- **Checkout**: Address select, payment method choose, order confirm.  
  - **Example**: Rahul address select karta hai, UPI se ₹1,59,998 pay karta hai (Orders, Addresses, Payments).  
- **Address Management**: Multiple addresses save karo, default set karo.  
  - **Example**: Rahul 2 addresses save karta hai, ek default set karta hai (Addresses).  
- **Order Tracking**: Status updates (Pending → Delivered).  
  - **Example**: Rahul ka order "Shipped" status pe update hota hai, tracking number milta hai (Orders).  
- **Cancellation**: Delivery se pehle cancel karo.  
  - **Example**: Rahul order cancel karta hai, stock vapis update hota hai (Orders, ProductVariants).  
**Validation**: Payment integration (Razorpay/Paytm) kaam karta hai, addresses soft delete hote hain.

#### 6. Discount System
**Models**: Coupons, CouponUsers, CouponRedemptions, DiscountRules  
**What**: Discounts aur promotions manage karo.  
**Flow**:  
- **Coupons**: User-specific ya general coupons.  
  - **Example**: Rahul "WELCOME20" coupon use karta hai, 20% discount milta hai (Coupons, CouponRedemptions).  
- **Bulk Discounts**: Retailers ke liye quantity-based discounts.  
  - **Example**: Amit 200 units order karta hai, 15% discount apply hota hai (DiscountRules).  
- **Promotions**: Time-bound offers.  
  - **Example**: Diwali sale mein 30% off, 5 din ke liye (DiscountRules).  
- **Redemption Tracking**: Coupon usage track karo.  
  - **Example**: Admin dekh sakta hai ki "WELCOME20" 100 baar use hua (CouponRedemptions).  
**Validation**: Coupons expiry aur usage limits ke saath validate hote hain, discounts real-time apply hote hain.

#### 7. Review System
**Models**: ProductReviews, Products, ProductVariants  
**What**: Verified reviews aur ratings.  
**Flow**:  
- **Reviews**: Sirf buyers reviews de sakte hain.  
  - **Example**: Rahul iPhone 15 ke liye 5-star rating aur "Great phone" review daalta hai (ProductReviews).  
- **Ratings**: Product page pe average rating dikhta hai.  
  - **Example**: iPhone 15 ka average rating 4.5 stars, 50 reviews (Products).  
- **Moderation**: Admins reviews approve/reject karte hain.  
  - **Example**: Admin fake review delete karta hai (ProductReviews).  
**Validation**: Sirf verified purchases (is_verified_purchase) reviews de sakte hain.

#### 8. Admin Dashboard
**Models**: Users, Orders, OrderItems, StockAlerts, Products, Categories, Brands, Coupons, DiscountRules, ProductReviews  
**What**: System ka complete control.  
**Flow**:  
- **User Management**: Users ban/unban, roles change.  
  - **Example**: Admin Rahul ko ban karta hai (Users).  
- **Analytics**: Sales, top products, revenue reports.  
  - **Example**: Admin monthly sales report dekhta hai, ₹10L revenue (Orders, OrderItems).  
- **Inventory**: Low-stock alerts monitor karo.  
  - **Example**: iPhone 15 ka stock 5 bachta hai, alert dikhta hai (StockAlerts).  
- **Content**: Products, categories, brands manage karo.  
  - **Example**: Admin "Samsung" brand add karta hai (Brands).  
- **Coupons & Discounts**: New offers create karo.  
  - **Example**: Admin "DIWALI30" coupon banata hai (Coupons).  
**Validation**: Dashboard se sab operations smooth hote hain, analytics real-time hote hain.

#### 9. Store Management
**Models**: Owners, Stores, Products  
**What**: Multi-vendor stores ke liye support.  
**Flow**:  
- **Owners**: Stores aur products manage karte hain.  
  - **Example**: Vikram "TechHub" store banata hai (Owners, Stores).  
- **Stores**: Products store-specific hote hain.  
  - **Example**: TechHub mein iPhone 15 add hota hai (Stores, Products).  
**Validation**: Store-specific products properly linked aur displayed hote hain.

---

### Technical Specifications
- **Backend**: Node.js + Express.js, PostgreSQL with Sequelize, JWT authentication, Razorpay/Paytm payments, Redis caching, RabbitMQ for async tasks (e.g., emails).  
- **Frontend**: React.js with Vite, Tailwind CSS for responsive UI.  
- **Integrations**:  
  - **Payment**: Razorpay/Paytm for payments/refunds.  
  - **Email**: SendGrid/Nodemailer for order confirmations, alerts.  
  - **Storage**: AWS S3 for media.  
- **Deployment**: AWS EC2/Heroku (backend), Vercel/Netlify (frontend), GitHub Actions for CI/CD, Prometheus/Sentry for monitoring.  

---

### Unique Selling Points (USPs)
1. **Dual Pricing**: Customers ke liye retail price, retailers ke liye bulk discounts (DiscountRules).  
2. **Smart Alerts**: Automatic low-stock alerts (StockAlerts) email/dashboard pe.  
3. **Custom Coupons**: Role-specific coupons (CouponUsers) aur redemption tracking.  
4. **Admin Power**: Ek dashboard se sab manage—users, products, orders, analytics.  
5. **Verified Reviews**: Sirf buyers ke reviews se trust (ProductReviews).  
6. **Multi-Vendor**: Owners apne stores aur products manage kar sakte hain (Stores).
